BACKGROUND

The following template should be provided in a tag activity when engaging a support engineer. This is to help keep the engineer informed so that they can pick back up with the investigation, and reduce the risk of repeated questions to the client.

STEPS

  1. Add Tag activity.
  2. Fill out the template below, enter into the tag activity - or e-mail to the tagged engineer if character limit reached.
  3. Re-assign ticket to engineer.

TEMPLATE


Section 1: Issue Description and Details:

  • Issue summary: (if differing from SR description or additional info added.)
  • Is reproducible?
    • Yes: (enter steps to reproduce)
    • No: (provide name of MTA, or reason why MTA cannot be collected)
    • Backend logs:
  • Sporadic? (if so, what frequency)
  • All users? (if specific, provide usernames)

Section 2: Investigation

  • Summary of investigation:
  • What's been ruled out:
  • Reason for tag:
  • IP engaged? (enter JIRA or Slack link here)

Section 3: Important info/updates

  • Any e-mails with important info: (add the e-mail date/time)
  • Add any data/screenshots to ticket, PHI in Secure Data Portal

Section 4: Urgency

  • Priority: (high/medium/low)
  • GoLive Impact: (yes/no)
  • Potential escalation: (yes/no)
