MatchListMgmt: Columns Sorting Not Saving

Back to Home

BACKGROUND

When attempting to sort/reorder columns, the order is not saving when refreshing the view. Notably, there is an empty column positioned first. This is caused due to corrupt application ini for the user, in particular the Pos=# for the column. Clearing the application for that section (Study List in this case) will resolve the issue.

ITEMS FOR INVESTIGATION

  • Username/UserID (person_id from PRSNL)
  • MTA (workflow)

STEPS

Query the application_ini table for the user and see what's defined for Pos=0.

select * from application_ini
where application_number=477601
and person_id = 21373771
and section = 'Study List'

If the application ini neds to be cleared, then perform the following:

Ensure the user is logged out of the application. If the app ini is cleared while the application is still open, it will just rewrite the same bad data as the application is closed since it's still running with bad settings in place.

Clear the application_ini for that section:

update into application_ini set parameter_data = ' '
where application_number=477601
and person_id = 21373771
and section = 'Study List'

Have the user log back in and retest.
